# Definitely Not Wordle

A deployed full-stack Wordle clone built with React, FastAPI, JWT auth, PostgreSQL persistence, and a public frontend/backend split.

## Live Demo

- **Frontend**: https://wordle-clone-ruddy-six.vercel.app
- **Backend**: https://wordle-clone-backend-se82.onrender.com
- **GitHub**: https://github.com/sjxchng/wordle-clone

## Architecture

**Frontend**
- React + TypeScript with Vite
- Hosted on Vercel
- Supports guest play, login/register modals, saved progress restore, physical keyboard input, and on-screen keyboard state
- Uses localStorage as a client-side fallback so guest play and in-browser progress remain usable if the backend is temporarily unavailable

**Backend**
- FastAPI service hosted on Render
- PostgreSQL database for users, saved game records, guess history, and stats
- JWT authentication with bcrypt password hashing
- Optional auth dependency so guests can play while logged-in users get persisted progress
- Daily shared answer generated deterministically from the current date
- Word validation through the Merriam-Webster Collegiate Dictionary API with a local word-list fallback

## Running Locally

Create a `backend/.env` file first:

```bash
DICTIONARY_API_KEY=your_merriam_webster_key
DATABASE_URL=your_postgres_connection_url
SECRET_KEY=your_jwt_secret
```

The `.env` file is ignored by Git and should not be committed.

**Backend**
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```

**Frontend**
```bash
cd frontend
npm install
npm run dev
```

The frontend runs on `http://localhost:5173` and the backend on `http://localhost:8000`.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/register` | Creates a new user account |
| POST | `/login` | Logs in and returns a JWT access token |
| GET | `/answer` | Returns the current secret word |
| POST | `/guess` | Accepts a 5-letter guess, returns per-letter feedback |
| GET | `/game` | Restores today's saved game for a logged-in user |
| GET | `/stats` | Returns completed-game stats for a logged-in user |

Guest players can use `/guess` without logging in. Logged-in guesses include the JWT token so the backend can save and restore progress.

## Implemented Behavior

- Public Vercel frontend talks to the public Render API.
- Guests can play without an account.
- Users can register, log in, and log out.
- Logged-in progress restores after refresh and after logging back in.
- Guest/browser progress is cached locally for refresh resilience.
- Logout clears the visible board and returns the user to guest mode.
- Result state is shown in a modal popup.
- How-to instructions are available in a modal.
- Invalid words return a user-facing error.
- Frontend and backend both handle stale or missing auth without blocking guest play.

## Deployment

- Frontend is hosted on Vercel.
- Backend is hosted on Render.
- The deployed backend uses PostgreSQL for users, saved guesses, and stats.
- Render environment variables provide `DATABASE_URL`, `SECRET_KEY`, and `DICTIONARY_API_KEY`.
- Vercel serves the static frontend bundle generated by Vite.
